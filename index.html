Q.7:- Write a function that accepts parameter n and returns factorial of n ?


        ANS:function factorial(n) {
                if (n === 0 || n === 1) {
                return 1;
                } else {
                return n * factorial(n - 1);
                }
                }
          
                console.log(factorial(5));
        OUTPUT: 120

Q.9:- Guess the output Abc(); const Abc = function(){ let value = 20; console.log(value); } 


        OUTPUT:In JavaScript, function declarations are hoisted to the top of their scope, but function expressions (such as the one assigned to Abc) are not hoisted. Therefore, when you try to invoke Abc() 
                before its declaration, you'll encounter a "TypeError: Abc is not a function" error.

Q.10:-Guess the output var a = 10; (function (){ console.log(a); var a = 20; })(); ?


        OUTPUT:In JavaScript, variable declarations are hoisted to the top of their scope, but not their assignments. This means that the var a = 20; statement inside the immediately-invoked function expression 
        (IIFE) is hoisted to the top of the function scope, but the assignment a = 10; remains in place.

        When the IIFE is invoked and tries to log the value of a, it refers to the a variable declared inside the function scope. At that point, the variable a has been declared but not yet assigned a value,
         so it holds the value undefined
